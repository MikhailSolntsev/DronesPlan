## Набросок UI

### Основной экран

- Верхняя панель:
  - Кнопка **«Синхронизировать сейчас»**.
  - Выбор периода: «Текущая неделя», «Прошлая неделя», произвольная дата.
  - Фильтры по сотруднику (assignee), статусу, типу задачи.

- Левая часть — **таблица задач**:

```text
| Priority | Key      | Summary                            | Notes | Jira |
|---------:|----------|------------------------------------|-------|------|
| 71       | PROJ-123 | Авторизация для мобильного клиента | text  | link |
| 61       | PROJ-120 | Оптимизация запросов               | text  | link |
| 51       | PROJ-110 | Сохранение черновиков              | text  | link |
| 40       | PROJ-130 | Улучшение логирования              | text  | link |
```

- Правила:
  - Сортировка по `Priority DESC`, затем по `Key`.
  - Цвет текста/плашки рядом может дублировать цвет зоны (blue/orange/grey).

### Правая часть — упрощённый календарь

Вариант 1: строки — **задачи**, столбцы — **дни**.

```text
             | 18.05 | 19.05 | 20.05 | 21.05 | 22.05 | 23-24.05 |
-------------+----+----+----+----+----+----+
PROJ-123     | Iv | Iv | Iv | Iv | Iv | ** |
PROJ-120     | Pe | Pe | Pe | Pe | Pe | ** |
PROJ-110     | Si | Si | Si | Si | Si | ** |
PROJ-130     |    |    |    |    |    | ** |
```

- Должны отображаться как рабочие дни недели, так и выходные. Должен быть показан календарь на 2 недели вперёд
- В ячейке показываем фамилию assignee (например, первые 2 буквы фамилии).
- Цвет ячейки:
  - синий — если приоритет относится к разработке (51/61/69/81);
  - оранжевый — если относится к тестированию (71/91);
  - серый/пусто — бэклог/релиз/закрыто.
- На уровне MVP состояние по дням может быть одинаковым (состояние последнего снапшота), без реконструкции истории на каждый день.
