## Интеграция с Jira (on-prem)

### Основные принципы

- Использовать REST API Jira, доступный во внутренней сети.
- Аутентификация: базовая (логин/пароль) или token — задаётся через конфигурацию, не хардкодится.

### Какие данные нужны

- Проект/фильтр определяется JQL.
- Для каждой задачи получаем:
  - `key`
  - `fields.summary`
  - `fields.status.name`
  - `fields.issuetype.name`
  - `fields.assignee.displayName` (или `name`, в зависимости от версии)
  - `self` или конструируемый URL на web-интерфейс Jira.

### Пример запроса

- Эндпоинт поиска:

```text
GET /rest/api/2/search?jql=<JQL>&fields=key,summary,status,issuetype,assignee
```

- Ответ содержит массив `issues`, каждый с нужными полями.

### Замечания

- Конкретный путь API (`/rest/api/2` или `/rest/api/3`) и детали аутентификации нужно будет уточнить по инстансу вашей Jira.
- Клиент `IJiraClient` должен быть спроектирован так, чтобы легко заменить реализацию (например, при переходе на облачную Jira).
